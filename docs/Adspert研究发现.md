# Adspert 绩效组和广告活动管理研究发现

## 核心概念

### 1. 广告活动导入和管理
- **自动导入**: 连接Amazon Ads账户后，Adspert会自动导入所有广告活动
- **选择性优化**: 用户可以决定哪些广告活动需要被优化，不必让Adspert优化所有广告活动
- **随时添加**: 可以随时将广告活动添加到优化中
- **即插即用**: Adspert是Plug & Play解决方案，连接后2分钟即可开始使用

### 2. 广告活动状态分类
根据研究，广告活动可以分为以下状态：

| 状态 | 描述 |
|------|------|
| **活跃 (Active)** | 正在运行的广告活动 |
| **暂停 (Paused)** | 用户手动暂停的广告活动 |
| **已介入优化 (Managed)** | 已被添加到绩效组，由算法自动优化 |
| **未介入优化 (Unmanaged)** | 未被添加到绩效组，不进行自动优化 |

### 3. 绩效组 (Performance Groups) 核心逻辑

#### 什么是绩效组？
- 绩效组是将多个广告活动组合在一起，设置统一优化目标的容器
- 可以跨账户、跨平台（如amazon.de, amazon.fr, ebay.de, google ads）优化
- 每个绩效组可以设置独立的优化目标（ACoS、ROAS、预算、利润最大化等）

#### 何时使用跨平台绩效组？
1. **单一配送中心**: 不关心销售来自哪个平台
2. **稳定的广告活动**: 有相同目标的成熟广告活动
3. **区域策略**: 想要基于整体表现分配广告支出
4. **多账户管理**: 同时管理Seller和Vendor账户

#### 何时不使用跨平台绩效组？
1. **表现不佳的广告活动**: 需要单独关注，创建独立绩效组
2. **新产品发布**: 避免被其他成熟产品抢占优化资源

### 4. 优化流程

```
1. 连接Amazon Ads账户
   ↓
2. 自动导入所有广告活动
   ↓
3. 选择要优化的广告活动
   ↓
4. 创建绩效组，分配广告活动
   ↓
5. 设置优化目标（ACoS/ROAS/预算/利润）
   ↓
6. 算法自动优化竞价
```

### 5. 支持的广告类型
- Sponsored Products
- Sponsored Brands
- Sponsored Display
- 自动和手动广告活动

### 6. 支持的市场
- 所有Amazon Ads市场
- eBay Advertising
- Google Ads
- Microsoft Ads
- Walmart Connect

## 对我们系统的启示

### 需要实现的功能

1. **广告活动自动导入**
   - 连接API后自动同步所有广告活动
   - 显示广告活动的原始状态（活跃/暂停）

2. **优化状态管理**
   - 已介入优化 (Managed): 已分配到绩效组
   - 未介入优化 (Unmanaged): 未分配到绩效组

3. **绩效组管理**
   - 创建绩效组并设置优化目标
   - 将广告活动分配到绩效组
   - 支持跨账户绩效组

4. **状态筛选器**
   - 按活跃/暂停筛选
   - 按已介入/未介入筛选
   - 组合筛选

### 界面设计建议

#### 广告活动列表页
- 显示所有导入的广告活动
- 状态标签：活跃/暂停 + 已介入/未介入
- 快速操作：加入绩效组、移出绩效组
- 批量操作支持

#### 绩效组管理页
- 绩效组列表
- 每个绩效组显示：名称、目标、包含的广告活动数、KPI
- 创建/编辑绩效组
- 管理绩效组内的广告活动
