# 亚马逊广告API研究文档

## 1. 自动广告四种匹配类型

根据亚马逊广告API文档，自动广告（Auto Targeting）支持四种匹配类型：

### 1.1 Close Match（紧密匹配）
- 向使用与产品紧密相关的搜索词的客户展示广告
- 适用于高度相关的关键词
- 通常转化率较高

### 1.2 Loose Match（宽泛匹配）
- 向使用与产品宽泛相关的搜索词的客户展示广告
- 覆盖范围更广
- 适用于发现新关键词

### 1.3 Substitutes（关联商品/替代品）
- 向浏览与您产品类似的商品详情页的客户展示广告
- 针对竞品的客户
- 适用于抢占竞品流量

### 1.4 Complements（同类商品/互补品）
- 向浏览与您产品互补的商品详情页的客户展示广告
- 针对购买相关产品的客户
- 适用于交叉销售

## 2. 竞价位置调整（Placement Bid Adjustments）

亚马逊广告API支持三种位置的竞价调整：

### 2.1 Top of Search (First Page)
- 搜索结果第一页顶部
- 最高可调整 +900%
- 曝光最佳位置

### 2.2 Product Pages
- 商品详情页
- 最高可调整 +900%
- 针对浏览商品的客户

### 2.3 Rest of Search
- 搜索结果中间或底部
- 2024年1月新增支持
- 最高可调整 +900%

## 3. 动态竞价策略（Dynamic Bidding）

### 3.1 Dynamic Bidding - Down Only
- 当广告不太可能转化时降低竞价
- 最多降低100%

### 3.2 Dynamic Bidding - Up and Down
- 根据转化可能性上下调整竞价
- 最多上调100%，下调100%

### 3.3 Fixed Bids
- 使用固定竞价
- 不受动态竞价影响
- 仅应用位置调整

## 4. 分时竞价（Dayparting / Schedule Bid Rules）

根据亚马逊广告API文档，Schedule Bid Rules功能支持：

### 4.1 功能特点
- 按小时、按天、按周设置竞价调整
- 支持特定日期范围
- 支持高流量事件（如Prime Day）

### 4.2 API支持
- 通过 Rules API 实现
- 支持 Sponsored Products
- 可设置竞价增加百分比

### 4.3 Amazon Marketing Stream
- 提供近实时的小时级报告
- 可用于分析不同时段的表现
- 支持基于历史数据的分时竞价优化

## 5. API端点总结

### 5.1 自动广告匹配类型
- 通过 targeting expressions 设置
- 可单独为每种匹配类型设置竞价

### 5.2 竞价位置调整
- Campaign By Placement API
- 支持 Top of Search 和 Product Pages
- 2024年新增 Rest of Search

### 5.3 分时竞价
- Schedule Bid Rules API
- 支持按时间设置竞价规则

## 6. 优化算法设计建议

### 6.1 自动广告匹配类型优化
1. 分析每种匹配类型的历史表现（ACoS、转化率、点击率）
2. 根据表现自动调整各匹配类型的竞价
3. 表现好的匹配类型提高竞价，表现差的降低或暂停

### 6.2 位置竞价优化
1. 分析不同位置的表现数据
2. 计算每个位置的最优竞价调整百分比
3. 高转化位置增加竞价，低转化位置降低竞价

### 6.3 分时竞价优化
1. 分析历史小时级数据
2. 识别高转化时段和低转化时段
3. 高转化时段提高竞价，低转化时段降低竞价或暂停
