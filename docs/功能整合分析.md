## 功能重叠分析

### 重叠1: 定时任务 vs 预算自动执行
- **定时任务**：可配置智能竞价调整的执行计划
- **预算自动执行**：专门用于预算建议的定时执行
- **结论**：功能重叠，应整合

### 重叠2: 定时任务 vs 数据同步
- **定时任务**：包含"数据同步"任务类型
- **数据同步**：专门的数据同步管理页面
- **结论**：数据同步应作为系统后台自动任务，不需要单独页面

### 重叠3: 自动回滚规则 vs 纠错复盘
- **自动回滚规则**：基于阈值自动建议回滚
- **纠错复盘**：检测归因延迟导致的错误
- **结论**：可整合到一个"自动纠错"模块

### 重叠4: 算法优化建议 vs 优化中心
- **算法优化建议**：算法参数调优
- **优化中心**：统一的优化系统
- **结论**：算法优化应作为优化中心的子功能

## 整合方案

### 方案：极简化设计（推荐）

#### 核心理念
> "算法自主决策执行，人只做监督"

#### 整合后的菜单结构

```
核心监控
├── 监控仪表盘    # 整体数据概览 + API连接状态
├── 健康度监控    # 广告活动健康状态
└── 预算预警      # 预算消耗预警

广告管理（算法自动决策执行）
├── 广告活动      # 查看/监督广告活动，显示最优出价建议
└── 绩效组管理    # 管理绩效组，查看利润最大化出价点

优化引擎（统一入口）
├── 优化中心      # 统一的自动优化系统（整合所有优化功能）
└── 季节性建议    # 大促活动预算调整（需要人工确认的特殊场景）

数据与报告
├── 竞价日志      # 出价调整记录
├── 出价调整历史  # 详细历史记录
├── 效果追踪报告  # 预估vs实际效果对比
└── 跨账号汇总    # 多账号数据汇总

系统设置
├── Amazon API    # API连接配置（含数据同步状态）
├── 优化设置      # 优化参数配置（含自动回滚规则、算法参数）
└── 通知设置      # 通知偏好

团队协作
├── 团队管理
├── 协作通知
└── 邮件报表
```

#### 删除/整合的页面

| 原页面 | 处理方式 | 整合到 |
|--------|----------|--------|
| 定时任务 /scheduler | 删除 | 后台自动运行，状态显示在仪表盘 |
| 数据同步 /data-sync | 删除 | 整合到Amazon API设置页面 |
| 预算自动执行 /budget-auto-execution | 删除 | 整合到优化设置 |
| 自动回滚规则 /auto-rollback | 删除 | 整合到优化设置 |
| 算法优化建议 /algorithm-optimization | 删除 | 整合到优化设置 |
| 智能预算分配 /intelligent-budget | 保留 | 整合到优化中心 |
| A/B测试 /ab-test | 删除 | 高级功能，暂时移除 |
| 纠错复盘 /correction-review | 保留 | 重要的人工审核功能 |

#### 自动化后台任务

以下任务完全在后台自动运行，无需人工配置：

1. **数据同步**：每2小时自动同步Amazon API数据
2. **智能竞价**：每天自动计算并应用最优出价
3. **健康度检查**：每天自动检测异常指标
4. **效果追踪**：自动收集7/14/30天效果数据
5. **自动回滚**：效果不佳时自动建议回滚（通知用户）
6. **N-Gram分析**：每周自动分析无效搜索词

#### 需要人工确认的场景

1. **季节性预算调整**：大促活动涉及大额预算变动
2. **纠错复盘**：归因延迟导致的错误需要人工判断
3. **批量回滚**：大规模回滚操作需要确认

### 第三步：后台自动化
- 确保所有定时任务在后台自动运行
- 在仪表盘显示自动化任务状态

### 第四步：清理冗余页面
- 保留路由但重定向到整合后的页面
- 删除不再需要的组件
