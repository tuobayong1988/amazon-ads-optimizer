## 系统概述

本系统是一款专为亚马逊卖家设计的智能广告优化平台，参考了Adspert.net等业界领先的广告优化工具的核心理念，结合亚马逊广告API的特性，实现了全面的自动化广告管理和优化功能。

### 支持的广告类型

| 广告类型 | 英文名称 | 主要用途 | 计费方式 |
|---------|---------|---------|---------|
| SP商品推广 | Sponsored Products | 推广单个商品，出现在搜索结果和商品详情页 | CPC |
| SB品牌广告 | Sponsored Brands | 推广品牌和多个商品，出现在搜索结果顶部 | CPC |
| SD展示广告 | Sponsored Display | 站内外展示广告，支持受众定向 | CPC/vCPM |

### 核心优化目标

系统支持四种核心优化目标，用户可根据业务阶段和需求选择：

1. **最大化利润**: 在保证利润率的前提下最大化绝对利润额
2. **目标ACoS**: 将广告销售成本比控制在指定范围内
3. **目标ROAS**: 确保广告投资回报率达到预期水平
4. **最大化销量**: 在预算范围内获取最多的订单量

## SP商品推广广告优化

SP广告是亚马逊最基础也是最重要的广告类型，本系统提供了全面的SP广告优化功能。

### 自动广告四种匹配类型优化

SP自动广告包含四种匹配类型，每种类型的优化策略有所不同：

| 匹配类型 | 英文名称 | 定向逻辑 | 优化重点 |
|---------|---------|---------|---------|
| 紧密匹配 | Close Match | 与商品高度相关的搜索词 | 转化率优化 |
| 宽泛匹配 | Loose Match | 与商品松散相关的搜索词 | 流量拓展 |
| 同类商品 | Substitutes | 浏览过类似商品的顾客 | 竞品流量获取 |
| 关联商品 | Complements | 浏览过互补商品的顾客 | 交叉销售 |

**优化算法核心逻辑**:

系统通过分析每种匹配类型的历史绩效数据，计算ACoS与目标值的比率，生成针对性的优化建议：

```
ACoS比率 = 实际ACoS / 目标ACoS

当 ACoS比率 > 1.5 时：建议降低竞价
当 ACoS比率 < 0.7 且 ROAS > 3 时：建议提高竞价获取更多流量
当 点击数 >= 最小点击阈值×2 且 转化数 = 0 时：建议暂停该匹配类型
```

### 广告位置竞价调整

亚马逊SP广告支持三个广告位置的差异化竞价：

| 广告位置 | 英文名称 | 特点 | 建议策略 |
|---------|---------|------|---------|
| 搜索结果顶部 | Top of Search | 曝光量大，转化率高，竞争激烈 | 高价值关键词重点投放 |
| 商品详情页 | Product Pages | 精准定向，转化率稳定 | 竞品定向策略 |
| 搜索结果其他位置 | Rest of Search | 成本较低，流量较大 | 流量拓展策略 |

**位置倾斜算法**:

系统基于市场曲线模型，计算每个位置的边际利润，自动调整位置竞价倍数（0-900%）：

```
位置利润 = 点击数 × (转化率 × 客单价 - CPC)
最优位置倍数 = argmax(位置利润)
```

### 分时竞价（Dayparting）

系统支持基于时间维度的精细化竞价控制：

**时间维度**:
- **星期维度**: 周一至周日，共7天
- **时段维度**: 每2小时一个时段，共12个时段
- **总控制单元**: 7 × 12 = 84个独立控制单元

**算法逻辑**:

系统分析历史数据中各时段的转化效率，生成时段竞价系数：

```
时段效率 = 时段转化率 / 平均转化率
时段竞价系数 = 基础竞价 × 时段效率调整因子
```

## SB品牌广告优化

SB品牌广告主要用于品牌曝光和多商品推广，优化策略与SP广告有所不同。

### 品牌广告类型

| 广告类型 | 英文名称 | 展示形式 | 适用场景 |
|---------|---------|---------|---------|
| 商品集合 | Product Collection | 品牌Logo + 多个商品 | 品牌认知建设 |
| 品牌旗舰店聚焦 | Store Spotlight | 品牌Logo + 旗舰店分类 | 引流到品牌旗舰店 |
| 视频广告 | Video | 视频 + 单个商品 | 高互动率场景 |

### 优化策略

**1. 竞价优化**

品牌广告的ACoS通常略高于SP广告，系统默认目标ACoS设置为30%：

```
建议竞价调整 = 当前竞价 × (1 - (实际ACoS - 目标ACoS) / 目标ACoS × 调整步长)
```

**2. 创意优化建议**

系统监控以下创意指标并生成优化建议：

| 指标 | 阈值 | 建议 |
|-----|------|-----|
| CTR | < 0.3% | 优化广告标题和图片 |
| CVR | < 5% | 检查落地页与广告一致性 |
| 视频观看率 | < 50% | 优化视频前3秒内容 |

**3. 新客户指标分析**

SB广告特有的新客户（New-to-Brand）指标对于品牌建设至关重要：

```
新客户价值 = 新客户订单数 × 客户终身价值系数
品牌广告真实ROI = (销售额 + 新客户价值) / 广告花费
```

| 兴趣定向 | Audience Interests | 特定兴趣标签的用户 | 人群扩展 |

### 浏览归因考量

SD广告的一个重要特点是支持浏览归因（View-through Attribution），即用户看到广告但未点击，之后仍然购买的情况：

```
有效ACoS = 广告花费 / (点击归因销售额 + 浏览归因销售额)
有效ROAS = (点击归因销售额 + 浏览归因销售额) / 广告花费
```

系统在评估SD广告效果时会综合考虑两种归因方式。

**计费方式切换建议**:

## 搜索词分析与优化

搜索词分析是广告优化的核心环节，直接影响广告的精准度和效率。

### 搜索词分类

系统将搜索词分为以下类别：

| 类别 | 判断条件 | 建议操作 |
|-----|---------|---------|
| 高价值词 | ACoS < 20% 且 转化数 >= 2 | 添加为精确匹配关键词 |
| 潜力词 | 有转化但ACoS略高 | 优化竞价或匹配方式 |
| 低效词 | ACoS > 50% 或 无转化 | 添加为否定关键词 |
| 数据不足 | 点击数 < 最小阈值 | 继续观察 |
| 品牌词 | 包含品牌名称 | 单独管理 |

### N-Gram词根分析

N-Gram分析是一种强大的搜索词分析技术，可以识别出表现不佳的词根组合：

**分析维度**:
- **Unigram（单词）**: 单个词的表现分析
- **Bigram（双词组合）**: 两个连续词的组合分析
- **Trigram（三词组合）**: 三个连续词的组合分析

**否定词根识别算法**:

```
对于每个N-Gram：
  如果 总转化数 = 0 且 总点击数 >= 最小点击阈值×2：
    标记为建议否定
  如果 平均ACoS > 低效阈值×1.5 且 总点击数 >= 最小点击阈值：
    标记为建议否定
```

### 搜索词迁移策略

从自动广告到手动广告的搜索词迁移是提升广告效率的重要手段：

**迁移流程**:

1. **识别高价值搜索词**: ACoS < 目标值 × 0.8 且 转化数 >= 2
2. **确定匹配类型**: 高价值词使用精确匹配，潜力词使用词组匹配
3. **设置初始竞价**: 基于历史CPC，高价值词提高10-15%
4. **在自动广告中否定**: 避免流量重复

## 市场曲线建模算法

市场曲线建模是本系统的核心算法之一，参考了Adspert的核心优化理念。

### 展现曲线建模

展现曲线描述了竞价与展现量之间的关系，通常呈对数函数形态：

```
展现量 = a × ln(竞价) + b

其中：
- a: 曲线斜率，反映市场竞争程度
- b: 截距项
```

### 点击率曲线建模

点击率与广告位置（由竞价决定）之间存在相关性：

```
CTR = c × 竞价^d + e

其中：
- c, d, e: 拟合参数
- 通常 d < 1，表示边际递减效应
```

### 利润曲线计算

综合展现曲线和点击率曲线，计算不同竞价下的预期利润：

```
利润(竞价) = 展现量(竞价) × CTR(竞价) × (CVR × AOV - 竞价)

其中：
- CVR: 转化率
- AOV: 平均订单价值
```

### 最优竞价点搜索

系统使用黄金分割搜索算法寻找利润最大化的竞价点：

```
function goldenSectionSearch(minBid, maxBid):
  φ = (1 + √5) / 2  // 黄金比例
  while (maxBid - minBid) > tolerance:
    bid1 = maxBid - (maxBid - minBid) / φ
    bid2 = minBid + (maxBid - minBid) / φ
    if profit(bid1) > profit(bid2):
      maxBid = bid2
    else:
      minBid = bid1
  return (minBid + maxBid) / 2
```

## 安全边界与风控机制

为确保自动优化的安全性，系统内置了多层风控机制。

### 竞价安全边界

| 边界类型 | 默认值 | 说明 |
|---------|-------|------|
| 最低竞价 | $0.10 | 防止竞价过低失去曝光 |
| 最高竞价 | $10.00 | 防止竞价失控 |
| 单次调整上限 | ±50% | 防止剧烈波动 |
| 日调整次数上限 | 1次 | 符合API限制 |

### 预算安全边界

| 边界类型 | 默认值 | 说明 |
|---------|-------|------|
| 最低日预算 | $5.00 | 确保基本曝光 |
| 预算调整上限 | ±30% | 防止预算剧烈变化 |
| 花费预警阈值 | 80% | 预算即将耗尽时预警 |

### 学习期机制

新广告活动或重大调整后，系统进入学习期：

| 学习阶段 | 时长 | 调整策略 |
|---------|-----|---------|
| 初始探索期 | 7天 | 仅收集数据，不做调整 |
| 数据积累期 | 7-14天 | 小幅调整（±10%） |
| 稳定优化期 | 14天后 | 正常优化 |

### 人工审核机制

以下操作需要人工审核后执行：

- 单次调整幅度超过30%的竞价变更
- 暂停高花费广告活动
- 批量否定关键词操作（超过10个）
- 预算调整超过日预算的50%
