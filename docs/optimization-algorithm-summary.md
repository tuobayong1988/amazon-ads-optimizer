## 一、算法体系总览

本文档整合了Amazon广告优化系统的完整算法体系，涵盖从数据采集到执行优化的全流程。系统采用**分层自动化**策略：核心优化模块采用**全自动执行+安全边界限制**，否词和迁移模块采用**算法分析+人工审核**。

### 1.1 算法模块架构

| 模块 | 核心功能 | 执行模式 | 安全边界 |
|------|----------|----------|----------|
| **预算控制** | 广告活动预算分配与调整 | 全自动执行 | ±25%（基于近7天平均花费） |
| **分时预算** | 基于周×时段的预算动态分配 | 全自动执行 | 按规则配置 |

| **分时竞价** | 基于周×时段的竞价动态调整 | 全自动执行 | 按规则配置 |
| **位置倾向** | 广告展示位置偏好设置 | 全自动执行 | ±25%（上限200%） |
| **搜索词判断** | 客户搜索词分类与处理 | 算法分析 | 人工审核 |

## 二、预算控制算法（全自动执行）

### 2.1 安全边界定义

> **核心约束**：不论上调预算还是下调预算，调整幅度不得超过**近7天平均花费的±25%**（排除最近2天数据）

**基准计算**：

```
基准花费 = AVG(第3天前 ~ 第9天前的每日花费)
最大上调 = 当前预算 × 1.25，且不超过 基准花费 × 1.25
最大下调 = 当前预算 × 0.75，且不低于 基准花费 × 0.75
```

### 2.2 预算分配算法

预算分配算法基于广告活动的历史表现，智能分配每日预算，最大化整体投资回报。

**核心指标权重**：

| 指标 | 权重 | 计算方式 |
|------|------|----------|
| ROAS | 40% | 销售额 / 花费 |
| 转化率 | 30% | 订单数 / 点击数 |
| ACoS | 20% | 花费 / 销售额（越低越好） |
| 点击率 | 10% | 点击数 / 曝光数 |

**分配公式**：

```
活动得分 = ROAS × 0.4 + CVR × 0.3 + (1 - ACoS) × 0.2 + CTR × 0.1
预算分配比例 = 活动得分 / 所有活动得分之和
建议预算 = 总预算 × 预算分配比例

// 安全边界校验
IF 建议预算 > 当前预算 × 1.25:
    最终预算 = 当前预算 × 1.25
ELIF 建议预算 < 当前预算 × 0.75:
    最终预算 = 当前预算 × 0.75
ELSE:
    最终预算 = 建议预算
```

### 2.3 预算消耗预警算法

监控预算消耗速度，识别异常消耗模式并发出预警。

| 预警类型 | 触发条件 | 预警级别 |
|----------|----------|----------|
| 消耗过快 | 当前消耗速度 > 预期速度 × 1.5 | 警告 |
| 消耗过慢 | 当前消耗速度 < 预期速度 × 0.5 | 提示 |
| 即将耗尽 | 剩余预算 < 2小时预期消耗 | 紧急 |

### 3.1 二维时段划分（7天 × 12时段 = 84个控制单元）

### 3.3 周×时段表现分析算法

```
数据采集周期：过去4周的历史数据

对于每个周×时段组合 [Di, Tj]：
  收集该组合在过去4周的表现数据
  
  时段ROAS[Di][Tj] = SUM(该组合销售额) / SUM(该组合花费)
  时段CVR[Di][Tj] = SUM(该组合转化数) / SUM(该组合点击数)
  时段CPC[Di][Tj] = SUM(该组合花费) / SUM(该组合点击数)
  
全周平均ROAS = 全周总销售额 / 全周总花费
表现系数[Di][Tj] = 时段ROAS[Di][Tj] / 全周平均ROAS
```

### 3.4 分时预算调整算法

```
对于当前时间点（周几=Di, 时段=Tj）：
  
  获取表现系数 = Performance[Di][Tj]
  
  IF 表现系数 > 1.5:
      预算调整 = +40% ~ +50%（超高价值时段）
  ELIF 表现系数 > 1.2:
      预算调整 = +20% ~ +40%（高价值时段）
  ELIF 表现系数 > 0.8:
      预算调整 = 0%（正常时段）
  ELIF 表现系数 > 0.5:
      预算调整 = -20% ~ -40%（低价值时段）
  ELSE:
      预算调整 = -40% ~ -50%（超低价值时段）
```

## 四、投放词竞价控制算法（全自动执行）

### 4.1 安全边界定义

> **核心约束**：不论上调竞价还是下调竞价，调整幅度不得超过**近7天平均CPC的±10%**（排除最近2天数据）

**基准计算**：

```
基准CPC = AVG(第3天前 ~ 第9天前的每日CPC)
最大上调 = 当前出价 × 1.10，且不超过 基准CPC × 1.10
最大下调 = 当前出价 × 0.90，且不低于 基准CPC × 0.90
```

### 4.2 竞价优化目标

根据优化目标（目标ACoS、目标ROAS、最大化销售）动态调整关键词出价。

**目标ACoS模式**：

```
当前ACoS = 花费 / 销售额
目标ACoS = 用户设定值

// 计算建议调整
IF 当前ACoS > 目标ACoS × 1.1:
    建议调整 = 降低 5% ~ 15%
ELIF 当前ACoS < 目标ACoS × 0.9:
    建议调整 = 提高 5% ~ 15%
ELSE:
    建议调整 = 0%（维持）

// 安全边界校验
最终调整 = CLAMP(建议调整, -10%, +10%)
```

**最大化销售模式**：

```
IF 转化率 > 行业均值 AND 曝光份额 < 80%:
    建议调整 = 提高 10% ~ 20%
ELIF 转化率 < 行业均值:
    建议调整 = 降低 5% ~ 10%

// 安全边界校验
最终调整 = CLAMP(建议调整, -10%, +10%)
```

### 4.3 竞价爬坡算法

对于新词或表现潜力词，采用渐进式出价提升策略。

```
爬坡阶段（每次调整受±10%限制）：
  阶段1（第1-3天）：起始出价 = 类目建议出价 × 0.8
  阶段2（第4-7天）：IF 有转化 THEN 出价 +10%（上限）
  阶段3（第8-14天）：IF ROAS达标 THEN 出价 +10%（上限）
  阶段4（第15天+）：进入正常优化模式
```

### 4.4 竞价纠错算法

检测因归因延迟导致的错误调价，并自动纠正（受±10%限制）。

```
纠错检测：
  1. 获取过去14天的调价记录
  2. 对比调价时的数据 vs 当前归因完成后的数据
  3. IF 调价时ACoS > 目标 但 归因后ACoS < 目标:
       标记为"误降价"，恢复出价（最多+10%）
  4. IF 调价时ACoS < 目标 但 归因后ACoS > 目标:
       标记为"误涨价"，降低出价（最多-10%）
```

## 五、分时竞价算法（全自动执行）- 周×时段二维控制

### 5.1 周×时段竞价系数

基于周×时段表现动态调整出价系数（受±10%总限制）。

```
竞价系数[Di][Tj] = 时段转化率[Di][Tj] / 全周平均转化率

高峰时段（系数 > 1.2）：出价 × 1.1（上限，受安全边界限制）
正常时段（系数 0.8-1.2）：出价 × 1.0
低谷时段（系数 < 0.8）：出价 × 0.9（下限，受安全边界限制）
```

### 5.2 周×时段竞价策略示例

```
周五晚间高峰（D5, T10-T11）：
  表现系数 = 1.7-1.8
  竞价调整 = +10%（达到安全边界上限）
  
周二凌晨低谷（D2, T1-T3）：
  表现系数 = 0.2-0.4
  竞价调整 = -10%（达到安全边界下限）
```

## 六、广告位置倾向算法（全自动执行）

### 6.1 安全边界定义

> **核心约束**：不论上调百分比还是下调百分比，调整幅度不得超过**近7天设定数据的±25%**（排除最近2天数据），且**上调不得突破200%的限制**

**基准计算**：

```
基准设定 = AVG(第3天前 ~ 第9天前的位置调整百分比)
最大上调 = MIN(当前设定 × 1.25, 200%)
最大下调 = 当前设定 × 0.75
```

### 6.2 位置表现分析

分析不同展示位置的表现，识别最优位置。

| 位置 | 典型特征 | 适用场景 |
|------|----------|----------|
| 搜索顶部 | CTR高，CPC高，转化率高 | 品牌词、高转化词 |
| 商品详情页 | CTR中，CPC中，关联购买 | 竞品词、互补品 |
| 其余位置 | CTR低，CPC低，曝光广 | 长尾词、探索性投放 |

### 6.3 位置调整算法

```
位置ROAS = 该位置销售额 / 该位置花费

// 计算建议调整
IF 搜索顶部ROAS > 其他位置ROAS × 1.3:
    建议调整 = +10% ~ +50%
IF 商品详情页ROAS > 其他位置ROAS × 1.3:
    建议调整 = +10% ~ +50%

// 安全边界校验
最终调整 = CLAMP(建议调整, -25%, +25%)
最终设定 = MIN(当前设定 + 最终调整, 200%)
```

## 七、客户搜索词判断算法（算法分析）

### 7.1 搜索词分类体系

| 分类 | 判断标准 | 处理方式 |
|------|----------|----------|
| **高相关-高转化** | 转化数≥2 且 CVR≥行业均值 | 迁移至精准匹配 |
| **高相关-低转化** | 点击数≥10 且 转化数=0 | 分析原因，考虑否定 |
| **低相关** | 与产品属性不匹配 | 添加否定词 |
| **竞品词** | 包含竞品品牌名 | 单独广告组管理 |
| **品牌词** | 包含自有品牌名 | 品牌保护组管理 |
| **长尾词** | 3个词以上的组合 | 评估后决定迁移 |

### 7.2 搜索词评估算法

```
搜索词评分 = 
  转化数 × 30 +
  (CVR / 行业CVR) × 25 +
  (1 - ACoS / 目标ACoS) × 25 +
  相关性得分 × 20

IF 评分 > 70:
    建议：迁移至精准匹配
ELIF 评分 > 40:
    建议：保持观察，继续收集数据
ELIF 评分 > 20:
    建议：降低出价或考虑否定
ELSE:
    建议：添加否定词
```

## 八、否词优化算法（人工审核）

### 8.1 算法流程

```
1. 数据采集
   - 导出搜索词报告
   - 筛选条件：点击数≥1 且 转化数=0

2. 词根拆分（Tokenization）
   - 将搜索词拆分为单字（Unigram）和双字组合（Bigram）
   - 示例："cheap blue running shoes" → 
     ["cheap", "blue", "running", "shoes", "cheap blue", "blue running", "running shoes"]

3. 核心词根排除（重要！）
   - 获取所有投放词的词根列表
   - 从分析结果中排除这些核心词根
   - 目的：避免过度否词导致丧失核心用户

4. 频率统计
   - 统计每个词根在"无效搜索词"列表中的出现频率
   - 计算公式：词根权重 = 出现次数 × 平均CPC

5. 阈值判定
   - 频率阈值：≥ 25次
   - 花费阈值：≥ $12.5
   - 满足任一条件即标记为"高频无效词根"

6. 生成建议（人工审核）
   - 输出建议否定的词根列表
   - 显示每个词根的出现频率、累计花费、影响搜索词数量
   - 等待人工审核确认
```

### 8.2 常见无效词根示例

| 词根类型 | 示例词根 | 否定原因 |
|----------|----------|----------|
| 价格敏感 | cheap, free, discount, budget | 低购买意向 |
| 二手/维修 | used, repair, fix, refurbished | 非新品需求 |

## 九、流量迁移算法（人工审核）

### 9.1 迁移漏斗架构

```
Tier 3（探索层）         Tier 2（验证层）         Tier 1（精准层）
┌──────────────┐        ┌──────────────┐        ┌──────────────┐
│ Broad/Phrase │  ──→   │    Phrase    │  ──→   │    Exact     │
│   匹配类型   │        │   匹配类型   │        │   匹配类型   │
└──────────────┘        └──────────────┘        └──────────────┘
     发现新词              验证转化                稳定收割
```

### 9.2 迁移触发条件

| 迁移路径 | 触发条件 | 附加操作 |
|----------|----------|----------|
| Broad → Phrase | 转化数≥2 且 CVR≥行业均值 | 原Campaign添加Negative Exact |
| Phrase → Exact | 转化数≥3 且 ROAS≥目标 | 原Campaign添加Negative Exact |
| 直接 → Exact | 高相关核心词（预设词库） | 所有Broad/Phrase添加Negative Exact |

### 9.3 流量隔离规则

```
原则：一个搜索词在同一时间只能在一个广告组中处于"激活"状态

隔离动作：
1. Tier 1 中的词 → 同步作为 Negative Exact 加入 Tier 2 和 Tier 3
2. Tier 2 中的词 → 同步作为 Negative Exact 加入 Tier 3
3. 发现冲突词时 → 保留表现最好的Campaign，其他添加否定
```

### 9.4 冲突消解算法

```
定期执行（每周）：

1. 扫描所有Campaign的Active Search Terms
2. 识别出现在2个及以上Campaign的搜索词

3. 对于每个冲突词：
   - 计算各Campaign中的表现指标（CVR、ROAS、ACoS）
   - winner = 表现最好的Campaign
   - losers = 其他所有Campaign
   
4. 生成建议（人工审核）：
   - 显示冲突词及其在各Campaign的表现
   - 建议保留的Campaign
   - 建议添加否定的Campaign
   
5. 人工确认后执行
```
