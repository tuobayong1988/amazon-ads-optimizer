## Adspert AI算法核心逻辑 ⭐⭐重要

### 算法三大核心组件

#### 1. 市场曲线 (Market Curves)
- **定义**：展示出价与点击数之间关系的图形表示
- **预测内容**：展示量、点击数、有效CPC
- **工作原理**：基于历史数据创建市场曲线，然后进行曲线拟合(Curve Fitting)
- **曲线拟合**：找到最佳拟合数据点的数学曲线，表示出价与表现之间的关系

#### 2. 决策树 (Decision Tree)
- **定义**：机器学习算法，通过树状结构将信息分割成更小的组
- **预测内容**：点击率(CTR)、转化率(CR)、转化价值(CV)
- **输入因素**：
  - 广告活动信息：名称、目标、预算、类型、标签
  - 关键词信息：匹配类型、长度、相似词
  - 产品信息：标题、品牌、产品组、库存、价格、重量/尺寸
- **特点**：通过机器学习不断构建决策树，形成"森林"

#### 3. 场景模拟 (Scenarios)
- **定义**：Adspert独创的预测模型，展示选定目标的可能结果
- **工作原理**：结合市场曲线、决策树数据、预算和绩效组目标约束
- **可视化**：曲线+阴影区域，阴影越窄表示目标越容易达成

### 出价优化决策过程
1. 检查所有可竞价对象的平均值
2. 选择平均能达成目标且符合预算的CPC出价
3. 同时最大化利润（利润 = 收入 - 广告支出）
4. 个别关键词可能不达标，但整体绩效组表现会达成目标

### 用户设置的两个关键约束
1. **绩效组目标 (Performance Group Goal)**
   - 设置在历史30天平均值的±20%范围内
   - 渐进式调整，不要一次性大幅改变
   
2. **广告活动预算 (Campaign Budgets)**
   - 确保表现好的广告活动有足够预算
   - 预算影响算法对广告活动重要性的判断

### 预算分配核心逻辑
- 如果高表现广告活动预算有限（如10€/天），但绩效组目标更高（如250€/天）
- 算法会将重点转移到有更多预算的其他广告活动
- **解决方案**：为高表现广告活动增加预算


## 智能预算分配算法设计 - 基于Adspert研究

### 核心设计理念

根据对Adspert后台的研究，智能预算分配的核心理念是**渐进式优化**：不会大幅调整单一广告活动的预算，而是根据数据表现一点一点优化，目的是让好的更好，让投产差的慢慢变好。

### 算法输入数据（近30天）

| 数据维度 | 说明 | 权重因子 |
|---------|------|---------|
| 日均花费 | 广告活动的平均每日消耗 | 基础指标 |
| 出单数量 | 广告带来的订单转化数 | 高权重 |
| 投产比(ROAS) | 销售额/广告花费 | 高权重 |
| ACoS | 广告花费/销售额 | 参考指标 |
| 点击率(CTR) | 点击数/展示量 | 中权重 |
| 转化率(CR) | 订单数/点击数 | 高权重 |

### 预算分配评分模型

**综合得分 = 转化效率得分 × 40% + ROAS得分 × 35% + 增长潜力得分 × 25%**

1. **转化效率得分**：(广告活动转化数 / 广告活动花费) / (绩效组平均转化效率)
2. **ROAS得分**：广告活动ROAS / 绩效组平均ROAS
3. **增长潜力得分**：基于预算利用率和边际效益曲线

### 预算调整规则

| 综合得分区间 | 预算调整幅度 | 说明 |
|-------------|-------------|------|
| > 1.3 | +5% ~ +10% | 高表现，适度增加预算 |
| 1.1 ~ 1.3 | +2% ~ +5% | 良好表现，小幅增加 |
| 0.9 ~ 1.1 | 0% | 正常表现，保持不变 |
| 0.7 ~ 0.9 | -2% ~ -5% | 表现欠佳，小幅减少 |
| < 0.7 | -5% ~ -10% | 表现较差，适度减少 |

### 约束条件

1. **单次调整上限**：每次调整不超过当前预算的10%
2. **最小预算保护**：不低于设定的最小日预算（如$5）
3. **总预算守恒**：绩效组内所有广告活动的预算调整总和为0
4. **冷却期**：同一广告活动两次调整间隔至少3天
5. **新广告保护期**：新建广告活动7天内不参与自动调整

### 预算再分配流程

```
1. 收集绩效组内所有广告活动的30天数据
2. 计算每个广告活动的综合得分
3. 识别需要减少预算的广告活动（得分 < 0.9）
4. 识别需要增加预算的广告活动（得分 > 1.1）
5. 计算可再分配的预算池（从低分广告活动释放）
6. 按得分比例分配预算池给高分广告活动
7. 应用约束条件，生成最终调整建议
8. 记录调整历史，用于效果追踪
```
